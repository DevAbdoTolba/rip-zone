## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, stroke widths, inline label padding |
| sm | 8px | Gap between Front/Back toggle buttons; gap between mode buttons |
| md | 16px | Padding inside toggle control container; map canvas margin from controls |
| lg | 24px | Vertical gap between controls block and SVG canvas |
| xl | 32px | Page horizontal padding on mobile |
| 2xl | 48px | Vertical padding above/below the muscle map section |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- **Touch targets:** Front/Back toggle buttons and detail mode buttons must meet 44px minimum touch target height. `size="sm"` Button from shadcn renders at 32px height — wrap with `min-h-[44px]` or use `size="default"` (40px) and accept 40px for pill buttons, adding 2px vertical padding to reach 44px.
- **SVG muscle hit targets:** Invisible overlay paths must cover a minimum 24×24 SVG coordinate units regardless of visual path size (brachialis, rotator cuff, supraspinatus). This is enforced in SVG authoring, not CSS.

## Interaction Contract

This section specifies exact interaction behaviors for the executor. There are no equivalents in upstream artifacts for these specifics — all are within Claude's discretion (CONTEXT.md).

### Front/Back Toggle (D-11, D-12, D-13)

- **Placement:** Above the SVG canvas, horizontally centered, full-width container with `max-w-xs mx-auto`
- **Structure:** Pill-shaped `role="group"` flex container with `bg-muted`, `border border-border`, `rounded-full`, `p-1`
- **Buttons:** Two shadcn `<Button>` with `size="sm"`, `rounded-full`, `capitalize`. Active state uses `variant="default"` (primary background). Inactive uses `variant="ghost"`
- **Behavior:** Instant swap on click — no transition or animation (D-12). Calls `useMapStore.setView()`
- **ARIA:** `aria-pressed={currentView === view}` on each button; `aria-label="Body view"` on group
- **Touch target:** Ensure each button has `min-h-[44px]` to meet touch target minimum

### Detail Mode Toggle (D-02, D-03)

- **Placement:** Directly below Front/Back toggle, 8px gap (`gap-sm`). Same horizontal centering
- **Structure:** Three-segment inline control — identical pill style to Front/Back toggle. Options: "Normal", "Advanced", "Anatomy"
- **Buttons:** shadcn `<Button>` with `size="sm"`, `rounded-full`. Active uses `className` override to force `--secondary` as background: `bg-secondary text-secondary-foreground`. Inactive uses `variant="ghost"`
- **Behavior:** Mode change is instant. Calls `useMapStore.setDetailMode()`. Disambiguation state (`zoomRegion`) resets to `null` on mode change
- **ARIA:** `aria-pressed` per button; `aria-label="Detail level"` on group
- **Note:** Mode toggle placement is above the SVG (stacked below Front/Back toggle), not to the side — mobile portrait screen space is narrow

### SVG Canvas Layout

- **Aspect ratio:** SVG viewBox `0 0 100 250` (portrait, 2:5 ratio — typical anatomical figure proportions). Width fills container, height auto-scales
- **Container:** `w-full max-w-sm mx-auto` — centered, capped at mobile width, responsive
- **Vertical layout (top to bottom):** Front/Back toggle → Detail mode toggle → SVG canvas
- **Gap:** 16px (`md`) between controls block and SVG canvas top edge; 8px (`sm`) between the two toggle rows

### Hover / Tap Feedback (D-10)

- **Mechanism:** CSS attribute selector in `globals.css` targeting `[data-view] path[id^="muscle-"]` — no per-path React state
- **Default fill:** `oklch(0.22 0.02 265)` on all muscle paths
- **Hover fill:** `oklch(0.32 0.05 265)` — applied via `:hover` pseudo-class. No transition duration (instant, matches D-12 snappy philosophy)
- **Cursor:** `cursor: pointer` on all interactive muscle paths
- **Selected fill:** `oklch(0.85 0.18 195)` (primary token) on the selected muscle path — applied via `[data-selected-muscle="<slug>"]` attribute on SVG wrapper
- **Non-interactive paths** (head, neck, hands, feet — D-09): `pointer-events: none`, fill `oklch(0.14 0.02 265)` (slightly lighter than background, clearly part of silhouette outline)

### Stroke Borders (D-07)

- **Normal mode paths:** `stroke-width: 1px`, stroke `oklch(0.18 0.03 265)`
- **Advanced mode paths:** `stroke-width: 1px`, stroke `oklch(0.18 0.03 265)` — same as Normal
- **Anatomy mode paths:** `stroke-width: 0.75px`, stroke `oklch(0.18 0.03 265)` — thinner because more paths share boundaries in tight anatomical regions

### Hit Targets (MAP-05, RESEARCH.md Pattern 2)

- **Implementation:** Two-layer SVG. Layer 1 (`id="visual-layer"`) contains filled, stroked visual paths. Layer 2 (`id="hit-layer"`) contains transparent paths with `fill="transparent"` and `stroke="none"`, positioned above Layer 1 in SVG stacking order
- **Minimum hit area:** Each hit path must cover at least 24×24 SVG coordinate units. For visual paths smaller than this, expand the hit path bounds while keeping the visual path accurate
- **Normal mode:** May use single-path with expanded `stroke-width` trick using `pointer-events: stroke` for simpler SVG authoring. Separate hit layer required for Advanced and Anatomy modes where paths are small and densely packed
- **Click handler:** Wired to hit-layer paths only. Visual paths have `pointer-events: none`

### Disambiguation Zoom (D-14–D-18)

- **Trigger:** In Advanced and Anatomy modes only (D-15). Normal mode: all muscle taps go directly to `selectMuscle()`. In Advanced/Anatomy: if tapped hit-path is in `CLUSTER_MAP`, call `setZoomRegion(CLUSTER_MAP[hitId])`
- **Zoom mechanism:** SVG `viewBox` attribute override — pass `zoomRegion.viewBox` as prop to SVGR component when `zoomRegion !== null` (RESEARCH.md Pattern: Disambiguation Zoom ViewBox Swap)
- **Dim overlay:** When `zoomRegion` is active, render a `<rect>` element in the SVG coordinate space covering the full original viewBox (`0 0 100 250`) with `fill="oklch(0.07 0.01 265 / 70%)"`. Position it above the visual layer but below the hit layer and label layer. Paths inside the cluster region will be above this rect in SVG stacking order
- **Labels:** SVG `<text>` elements at each muscle's centroid (x, y in SVG coords). Font `fontSize="6"`, `fill="oklch(0.95 0.01 270)"`, `textAnchor="middle"`. Render only when `zoomRegion !== null`
- **Dismiss:** Tapping a specific muscle in zoom view calls `selectMuscle(slug)` then `setZoomRegion(null)` — map returns to full body (D-16). Tapping outside the cluster region also calls `setZoomRegion(null)` (via a backdrop rect with `onClick`)
- **CLUSTER_MAP and centroids:** Defined after SVG authoring is complete — left as empty `{}` initially. All muscle taps resolve directly to `selectMuscle()` until CLUSTER_MAP is populated
- **Animation:** No zoom animation. ViewBox change is instant (consistent with D-12 snappy philosophy)
