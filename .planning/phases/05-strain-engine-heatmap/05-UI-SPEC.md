### Strain Heatmap Color Palette (Phase 5 specific, locked in CONTEXT.md)

| StrainLevel | Color Value | Visual Role |
|-------------|-------------|-------------|
| Rested | `oklch(0.22 0.02 265)` (default gray) | No inline override applied — CSS default maintains gray |
| Light | `#3b82f6` (blue) | Muscle recently trained, low accumulation |
| Moderate | `#eab308` (yellow) | Muscle trained with moderate volume |
| Heavy | `#f97316` (orange) | Muscle trained with high volume |
| Strained | `#ef4444` (red) | Muscle at or near full strain capacity |

**Implementation note:** Strain colors are applied via `style.fill` inline DOM mutation on SVG path elements (same pattern as MiniMuscleMap — Phase 3 decision). They do NOT use CSS custom property variables because CSS cannot resolve dynamic property names in static selectors. The inline style approach has highest CSS specificity priority.

**Selected muscle override:** When a muscle is both selected AND strained, the strain `useEffect` skips that slug (`if (slug === selectedMuscle) continue`). The CSS `data-selected` rule applies the primary accent fill instead. Selection state always wins over strain color.

**Source:** CONTEXT.md "Heatmap Color Palette & Rendering" (locked decision) + RESEARCH.md STRAIN_COLORS constant.

## Interaction Contract

### Strain Heatmap Rendering

| Interaction | Behavior |
|-------------|----------|
| Page mount | `useStrainMap()` hook loads Dexie history, computes strain, applies fills via `useEffect` |
| Workout session ends | `activeSession?.id` dependency change triggers `useStrainMap` recalculation |
| Front/Back view toggle | Strain `useEffect` re-fires (includes `currentView` in deps) — fills re-applied to new SVG DOM nodes |
| Detail mode switch (Normal/Advanced/Anatomy) | Strain `useEffect` re-fires (includes `detailMode` in deps) — fills re-applied |
| Muscle selected | Selected muscle skipped in strain fill loop — primary accent color takes precedence |
| Muscle deselected | On next strain effect run, deselected muscle gets strain color if non-Rested |
| Rested muscle | No inline `style.fill` applied — CSS default `oklch(0.22 0.02 265)` handles it |
| Anatomy mode bilateral muscles | Apply color to both `#muscle-{slug}-left` and `#muscle-{slug}-right` paths |
| Active session in progress | No recalculation during session — `activeSession?.id` is stable string for entire session |

### Hover States

Existing Phase 2 CSS hover rule `fill: oklch(0.32 0.05 265)` applies to all muscle paths. With inline `style.fill` override on strained muscles, the hover will NOT override the strain color because CSS hover cannot override inline styles. This is acceptable — strained muscles visually indicate their state without hover brightening. No change to existing CSS needed.

### No Polling

The strain map does NOT update on a timer. It updates only on: (1) page mount and (2) `activeSession?.id` change (session start or end). This is a locked decision from CONTEXT.md.

### Strain Color Rendering on SVG

The muscle map SVG paths receive `style.fill` directly via DOM mutation in a `useEffect`. No changes to the SVG files or globals.css are required for color application.

```
StrainLevel.Light    → path.style.fill = '#3b82f6'  (blue)
StrainLevel.Moderate → path.style.fill = '#eab308'  (yellow)
StrainLevel.Heavy    → path.style.fill = '#f97316'  (orange)
StrainLevel.Strained → path.style.fill = '#ef4444'  (red)
StrainLevel.Rested   → no inline fill set (CSS default gray maintained)
```
