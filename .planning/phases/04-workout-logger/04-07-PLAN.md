<objective>
Fix the PR badge never-fires bug in WorkoutLogger.tsx

The root cause: `loadHistoricalSets` queries ALL Dexie sets (including the just-confirmed set from the current session). After `confirmSet` writes to Dexie, the `useEffect` re-runs `loadHistoricalSets`, which now includes the new set. `isNewPR` uses strict `>`, so `100 > 100 = false` — the badge never appears.

Fix: Replace the reactive `useEffect`-based historicPRs loading with a snapshot approach. When an exercise is added to the session, load its historical PRs from Dexie ONCE and freeze that snapshot in a `sessionPRBaseline` ref. All subsequent `isNewPR` checks compare against this frozen baseline, not the live Dexie state that includes current-session writes.

Purpose: Unblocks Success Criterion 4 (WORK-04) — "User sees a visual callout highlighting a new personal record at the moment it is set during a session"
Output: Fixed WorkoutLogger.tsx with snapshot-based PR detection, plus regression test
</objective>

       ```typescript
       async function loadHistoricalSets(
         exerciseSlug: string,
         excludeSessionId: string | null
       ): Promise<Pick<SetLogRecord, 'reps' | 'weightKg'>[]> {
         try {
           const { workoutsDb } = await import('@/lib/db/workouts')
           const exerciseRecords = await workoutsDb.exercisesInSession
             .where('exerciseSlug')
             .equals(exerciseSlug)
             .toArray()
           // Filter out exercises belonging to the current session
           const historicalRecords = excludeSessionId
             ? exerciseRecords.filter(r => r.sessionId !== excludeSessionId)
             : exerciseRecords
           const allSets: Pick<SetLogRecord, 'reps' | 'weightKg'>[] = []
           for (const record of historicalRecords) {
             const sets = await workoutsDb.sets
               .where('exerciseInSessionId')
               .equals(record.id)
               .toArray()
             allSets.push(...sets.map(s => ({ reps: s.reps, weightKg: s.weightKg })))
           }
           return allSets
         } catch {
           return []
         }
       }
       ```

       const sessionPRBaseline = useRef<Map<string, Map<number, number>>>(new Map())
       const loadedSlugs = useRef<Set<string>>(new Set())

       useEffect(() => {
         const loadNewBaselines = async () => {
           const sessionId = activeSession?.id as string | null
           for (const ex of activeExercises) {
             const slug = ex.exerciseSlug as string
             if (!loadedSlugs.current.has(slug)) {
