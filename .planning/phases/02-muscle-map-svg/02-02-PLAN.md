    Author two SVG files with these specifications:

    **Shared SVG structure for both files:**
    - `viewBox="0 0 100 250"` (portrait, 2:5 ratio per UI-SPEC)
    - `xmlns="http://www.w3.org/2000/svg"`
    - No `width`/`height` attributes (component will size via CSS)
    - Three `<g>` layer groups in this stacking order:
      1. `<g id="outline-layer">` — non-interactive body silhouette (head, neck, hands, feet)
      2. `<g id="visual-layer">` — interactive muscle paths with fills and strokes
      3. `<g id="hit-layer">` — transparent hit-target overlay paths

    **Non-interactive outline paths** (in `<g id="outline-layer">`):
    - Head: `<ellipse id="outline-head" .../>` — centered at x=50, roughly y=12, rx=8, ry=10
    - Neck: `<rect id="outline-neck" .../>` or `<path>` — narrow connector between head and shoulders
    - Hands (left/right): `<path id="outline-hand-left" .../>`, `<path id="outline-hand-right" .../>`
    - Feet (left/right): `<path id="outline-foot-left" .../>`, `<path id="outline-foot-right" .../>`
    - All outline paths: `fill="oklch(0.14 0.02 265)"`, `stroke="oklch(0.18 0.03 265)"`, `stroke-width="1"`

    **Visual muscle paths** (in `<g id="visual-layer">`):
    - Each path: `id` = svgRegion value from muscles.json (e.g., `id="muscle-pectoralis-major"`)
    - Default fill: `fill="oklch(0.22 0.02 265)"`
    - Stroke: `stroke="oklch(0.18 0.03 265)"`, `stroke-width="1"`
    - `pointer-events="none"` (clicks go through to hit layer)

    **Hit target paths** (in `<g id="hit-layer">`):
    - Each path: `id` = `hit-` prefix + muscle svgRegion (e.g., `id="hit-muscle-pectoralis-major"`)
    - `fill="transparent"`, `stroke="none"`
    - Minimum 24x24 SVG coordinate units coverage
    - Same shape as visual path but can be expanded for small muscles
